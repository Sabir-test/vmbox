# playbooks/

Step-by-step runbooks for common VM tasks. Written in Markdown so they can
be read by humans or executed by AI agents.

## Conventions

- One file per procedure
- Filename: `<verb>-<noun>.md` (e.g., `rebuild-dev-base.md`, `reset-gpu-driver.md`)
- Each file starts with a **When to use** section and ends with a **Verify** section
- Commands must be copy-pasteable and tested
